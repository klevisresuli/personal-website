Smart Traffic Control System

April 2024 - May 2024

The idea is creating a system based on ESP32-Cam that is going to control the traffic light based on the number of cars on each side of an intersection making the traffic easier and also giving priority to emergencies(case of an ambulance being in the intersection).

  • main code is a proof of concept that does calibration, undistortion, pose estimation and homography to estimate positions and directions of cars as shown by the green dots on the left picture. For deployment in a stationary environment (camera is stationary in world frame), only the position and direction estimation would need to be done during inference.
  • ESP32-CAM firmware consists of two threads running in parallel. The first one runs the handleClients where ESP32 works as a server and we can get a capture from the Python script using the dialog window showing the camera live-stream and pressing the k button on the keyboard. The second thread handles the traffic light control. First, it reads the data.json file from the git repo and deserializes it into JSON format. After that, we read the specific direction and adjusted the green light timing accordingly as presented on the right picture. For more details check the code.

The Elevator Project

January 2024 - April 2024

A software for controlling 3 elevators,hardware shown in the left figure, working in parallel across 4 floors using Go coding language.

The system ensures efficient elevator service, even in the presence of failures such as network disconnections or power loss. The design uses a peer-to-peer network topology, where each elevator (node) communicates via UDP broadcasts to make real-time decisions on servicing calls. The core of the system is a finite state machine (FSM) that governs each elevator‚Äôs operation as shown in the diagram on the right. Each elevator independently processes requests, utilizing a cost function to prioritize calls while ensuring no requests are lost. In case of a failure, a backup process takes over to continue elevator operation and recover pending tasks without user intervention.

Key features include:

  • Fault tolerance: The system handles elevator disconnections and failures gracefully, ensuring no service interruptions.
  • Concurrency: Go‚Äôs concurrency model was leveraged to ensure smooth, parallel operation of elevators.
  • Cost efficiency: Calls are distributed efficiently among elevators based on their current state, using the cost function to minimize wait times.

Ping Pong Game

September 2023 - November 2023

Part of a term project in TTK4155 course at NTNU, where a distributed embedded control system was built.

  • Developed a distributed embedded control system for a Ping Pong game as part of the TTK4155 course at NTNU, utilizing CAN protocol for communication between nodes.
  • Designed and implemented Node 1 using an ATMEGA162 microcontroller integrated with a USB gaming board, CAN interface, SRAM, and A/D converter for precise data acquisition.
  • Constructed Node 2 with an Arduino Due development board, custom I/O shield, motorbox, servo, and solenoid to control the physical movements of the Ping Pong game.
  • Achieved accurate and responsive control of the Ping Pong system, enhancing gameplay experience by ensuring minimal latency between user input and motor responses.
